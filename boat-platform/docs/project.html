Features

What BoAt provides

BoAt is designed for OEM engineers, Tier-1 suppliers, and open-source contributors who need deterministic, high-throughput simulation with direct hardware integration.

⚙️
Deterministic Simulation Core
Tick-based scheduler with a seeded determinism engine guarantees bit-identical results across runs and environments. Ideal for CI regression testing.
🔌
Stable Plugin SDK
C ABI v7 plugin interface with C++ convenience wrappers. Plugins expose initialize, on_tick, on_can_frame, and shutdown hooks. Load shared objects at runtime without recompilation.
📡
gRPC API Gateway
Full gRPC service surface: SimulationService, SignalService, ScenarioService, ReplayService, FaultService, MetricsService, TraceService, CanService, and the always-on BusService.
🚌
BoAt Signal Bus
Always-on named typed signal pub/sub — independent of simulation lifecycle. Nodes publish numbers, strings, booleans, or raw bytes. Zero-config inter-node communication over gRPC streaming.
🔧
CAN & Ethernet HIL
Virtual (vcan) and physical SocketCAN interfaces. Multi-bus registry with per-interface frame subscription. Virtual Ethernet interfaces backed by UDP multicast — each named veth maps to an isolated multicast group and port. Always-on node plugins wired at gateway startup, independent of simulation state.
🐍
Python SDK
CanNode and BusNode base classes for writing nodes in pure Python. Background threading built-in. Full gRPC stub coverage for all services via BoAtClient.
📼
Event Store & Replay
SQLite-backed event store and flat-file trace store. Deterministic replay controller reconstructs any prior simulation run with identical signal sequences.
⚡
Fault Injection
Seeded deterministic fault injector for reproducing fault scenarios. Inject signal errors, CAN bus dropouts, and timing faults on-demand via the FaultService API.
🖥️
Live Monitor Dashboard
Port 8080: real-time CAN frame trace, BoAt bus signal log, and event log — all filterable and pauseable. Port 8081: system overview showing active components and connections.
🛠️
CLI Tool (boat-cli)
Command-line interface for simulation control, scenario management, signal inspection, CAN bus listing, plugin operations, replay control, and metrics queries.
📦
Scenario Loader
TOML-based scenario definitions loaded at runtime. Scenarios configure initial signal state, plugin selection, fault schedules, and simulation duration.
🔄
CI/CD Ready
CMake presets for debug, release, ASAN, and TSAN builds. GitHub Actions workflows for build, test, and release. Docker images for dev and runtime environments.
Architecture

System overview

BoAt uses a layered architecture. Clients communicate over gRPC. The gateway delegates to the C++ simulation core and plugin layer. Hardware is abstracted behind a HAL. All state is persisted in the storage layer.

Requirements

System requirements

BoAt builds and runs on Linux. Windows and macOS are not supported (SocketCAN and vcan are Linux-only).

🖥️ Operating System

  • Ubuntu 22.04 LTS or newer (recommended)
  • Any Linux distribution with kernel ≥ 5.10
  • vcan kernel module for virtual CAN

🔨 Build Tools

  • CMake ≥ 3.24
  • Ninja build system
  • GCC ≥ 12 or Clang ≥ 15 (C++20 required)
  • git (for FetchContent dependencies)

🐍 Python

  • Python ≥ 3.11
  • grpcio ≥ 1.80.0
  • grpcio-tools ≥ 1.80.0
  • protobuf ≥ 6.31.1
  • fastapi + uvicorn (dashboards only)

🦀 Rust (for IPC layer)

  • rustup / cargo (stable toolchain)
  • Required by iceoryx2 shared-memory IPC
  • Install: curl https://sh.rustup.rs | sh

📦 C++ Dependencies

  • gRPC + Protobuf — fetched automatically via CMake FetchContent
  • spdlog — fetched automatically
  • tomlplusplus — fetched automatically
  • iceoryx2 (C FFI) — fetched automatically
  • SQLite3 — system package (libsqlite3-dev)

💾 Hardware (optional)

  • Physical CAN interface (e.g. Peak PCAN, Kvaser) for HIL
  • Without hardware: vcan0 / vcan1 virtual interfaces
  • Minimum 4 GB RAM for full build (gRPC compilation)
  • 8 GB RAM recommended
Installation

Getting started

Follow these steps to build BoAt from source and run the gateway. The first build downloads all C++ dependencies — expect 10–20 minutes depending on your internet connection.

1
Install system dependencies
Install the build tools, Python, SQLite, and the Rust toolchain.
# Ubuntu / Debian
sudo apt-get update && sudo apt-get install -y \
    build-essential cmake ninja-build git \
    python3 python3-pip libsqlite3-dev \
    linux-modules-extra-$(uname -r)   # vcan module

# Rust (required for iceoryx2 IPC layer)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
2
Clone the repository
git clone https://github.com/your-org/boat-platform.git
cd boat-platform
3
Build the C++ gateway and core
CMake FetchContent downloads gRPC, Protobuf, spdlog, and iceoryx2 on first configure. Use the debug preset for development or release for production. 
# Configure (debug)
cmake --preset debug

# Build everything
cmake --build build/debug

# Or release build
cmake --preset release && cmake --build build/release
First build note: Downloading and compiling gRPC takes 10–20 minutes. Subsequent builds are incremental and fast.
4
Install the Python SDK and CLI
# SDK
pip install -e sdk/python

# CLI
pip install -e cli

# Dashboard dependencies
pip install fastapi uvicorn
5
Set up virtual CAN interfaces
Required for running nodes and the dashboard without physical hardware.
# Load the vcan kernel module
sudo modprobe vcan

# Create virtual interfaces
sudo ip link add dev vcan0 type vcan
sudo ip link set up vcan0

sudo ip link add dev vcan1 type vcan
sudo ip link set up vcan1
To make these persistent across reboots, add the commands to /etc/rc.local or create a systemd service.
6
Start the gateway
The gateway binds to 0.0.0.0:50051. Set BOAT_CAN_INTERFACES to the comma-separated list of CAN interfaces to open. Optionally load always-on C++ node plugins with BOAT_NODE_PLUGINS. 
# Basic — opens vcan0
BOAT_CAN_INTERFACES=vcan0 ./build/debug/src/gateway/grpc_gateway/boat_gateway

# Multiple interfaces + an always-on C++ node plugin
BOAT_CAN_INTERFACES=vcan0,vcan1 \
BOAT_NODE_PLUGINS=./build/debug/plugins/can_responder/libcan_responder_plugin.so \
./build/debug/src/gateway/grpc_gateway/boat_gateway
Usage

How to use BoAt

With the gateway running, interact via the CLI, Python SDK, dashboards, or your own nodes.

🛠️ CLI — simulation control
# Start a simulation
boat sim start --scenario scenarios/default.toml

# Check status
boat sim status

# Stop
boat sim stop

# List loaded plugins
boat plugin list

# List CAN buses
boat can list-buses
📡 CLI — signals and replay
# Watch live signals
boat signal watch vehicle.speed engine.rpm

# Set a signal value
boat signal set vehicle.speed 120.5

# List recent trace events
boat trace list --limit 20

# Replay a recorded session
boat replay start --session session-001
🐍 Python — CAN node
from boat.can_node import CanNode

class MyNode(CanNode):
    def on_frame(self, frame, iface):
        print(f"{iface}  0x{frame.can_id:03X}"
              f"  {frame.data.hex(':')}")

node = MyNode(iface_filter="vcan0")
node.run()
🚌 Python — Bus signal pub/sub
from boat.bus_node import BusNode

# Publisher
pub = BusNode(node_id="sensor-sim")
pub.publish("vehicle.speed",  120.5)
pub.publish("engine.rpm",     3200)
pub.publish("door.lock",      True)

# Subscriber
class Monitor(BusNode):
    def on_signal(self, sig):
        print(sig.name, sig.number_value)

m = Monitor()
m.run(names=["vehicle.speed"])
🔄 Python — Cyclic sender node
# Start the cyclic sender
python3 nodes/cyclic_sender_node.py

# In another terminal —
# send 0x111 to start cyclic output
cansend vcan0 111#DEADBEEF

# Update the payload at runtime
python3 nodes/payload_configurator_node.py
> cyclic AABBCCDD

# Send 0x112 to stop
cansend vcan0 112#00
🖥️ Dashboards
# Live Monitor — CAN trace + Bus signal log
python3 demo/dashboard.py
# → http://localhost:8080

# System Overview — components & connections
python3 demo/system_dashboard.py
# → http://localhost:8081

# Custom port
BOAT_DASH_PORT=9090 python3 demo/dashboard.py
⚡ Fault injection
# Via CLI
boat fault inject \
  --signal vehicle.speed \
  --type noise --magnitude 5.0

# Via Python SDK
from boat.client import BoAtClient
from boat.v1 import fault_pb2

c = BoAtClient()
c.fault.InjectFault(
  fault_pb2.InjectFaultRequest(
    signal_name="vehicle.speed",
    fault_type="noise",
    magnitude=5.0))
🏗️ Writing a C++ plugin
// my_plugin.cpp
#include <boat/plugin.h>

static BoatPluginVTable kVTable = [] {
  BoatPluginVTable vt{};
  vt.initialize = [](void*, const char*) {
    return 1;
  };
  vt.on_tick = [](void*, uint64_t tick) {};
  vt.shutdown = [](void*) {};
  return vt;
}();

extern "C" {
  const BoatPluginVTable* boat_plugin_vtable() {
    return &kVTable;
  }
  int boat_plugin_abi_version() {
    return BOAT_PLUGIN_ABI_VERSION;
  }
}
Environment variables summary:
BOAT_CAN_INTERFACES — comma-separated CAN interfaces opened at gateway startup (default: vcan0)
BOAT_NODE_PLUGINS — comma-separated .so paths for always-on C++ node plugins
BOAT_DASH_PORT — override dashboard port (default: 8080)
